Git Commands!

Git Commando Beskrivning Användning
git cherry-pick [commit] Tar en specifik commit från en annan gren och applicerar den på den aktuella grenen Används för att välja en enskild commit från en annan gren och applicera den på den aktuella grenen. Det är användbart när du vill inkludera en specifik ändring från en annan gren utan att behöva sammanfoga hela grenen
git rebase [branch] Flyttar de lokala ändringarna till basen av en annan gren Används för att omorganisera commit-historiken genom att flytta dina lokala commits till basen av en annan gren. Det används ofta för att hålla commit-historiken ren och linjär särskilt vid arbete med grenar som har delats med andra.
git blame [fil] Visar vem som senast ändrade varje rad i en fil och vilken commit som gjorde ändringen Används för att spåra vem som gjorde varje ändring i en specifik fil genom att visa vilken commit som introducerade varje rad.
git grep [pattern] Söker efter mönster i filerna i arbetsträdet Används för att söka efter ett visst mönster i filerna i arbetsträdet. Detta är användbart för att snabbt hitta förekomster av specifikt innehåll i projektet.
git bisect [bad] [good] Används för binär sökning för att hitta en specifik commit som orsakade en förändring i beteendet mellan två tillstånd Används för att genomföra en binär sökning för att identifiera den specifika commit som orsakade en förändring i beteendet mellan två kända tillständ, en "dålig" och en "bra" commit.
git describe Ger en beskrivning av den närmaste taggen till en specifik commit. Ger en beskrivning av den närmaste taggen till en specifik commit, vilket kan vara användbart för att förstå var en viss commit befinner sig i förhållande till taggar i projektet.
