Git Commands!

Git Commando Beskrivning Användning
git rm [file] Tar bort en fil både från din arbetskatalog och Git-indexet Används om du vill ta bort en fil helt och hållet från Git och din lokala dator
git mv [gammalFil] [nyFil] Flytta eller ge nytt namn till en fil Används om du vill byta namn på en fil, tar bort den gamla och lägger till den nya filen i git-indexet
git mergetool Öppnar ett verktyg för att hantera en konflikt visuellt vid en merge Används om det uppstår en konflikt vid en merge. Efter konflikten är löst kan man fortsätta sin merge
git fetch [remote] Hämtar nya ändringar från det angivna fjärrrepositoryt, men ändrar inte i din lokala Används för att kunna granska ändringar och välja vad du vill använda
git show Visar information om den senaste commiten, (kan även lägga till ett commit-ID för att se en specifik commit) Används för att granska förändringar och se historiken i ett Git-repository
git tag Listar alla taggar som finns i ett repository Taggar används för att spåra specifika eller viktiga punkter i ett projekt.
git stash Lagrar temporärt ändringar lokalt som ännu inte har blivit commitade Används när du behöver dra en pull, men inte förlora det senaste du arbetat på. Eller jobbar med flera uppgifter elelr behöver byta branch.
git stash pop Återapplicerar den senaste stashen och tar bort den från stash-stacken Används tex efter man har gjort en pull och vill fortsätta arbeta med sin kod.
git clean Tar bort ocommittade filer från arbetskatalogen Används när man vill rensa arbetskatalogen från filer man inte längre behöver spåra med git.
git revert [commit] Skapar en ny commit som inverterar ändringar gjorda i den specifika commiten man vill ångra. Används när man vill ångra en commit och bevara historiken.
git config --list Listar alla Git-konfigurationsvariabler och deras värden. Används när man vill kontrollera sina Git-konfigurationer och se att de har ställts in korrekt.
git gc Samlar ihop onödiga filer och komprimerar eller tar bort Git-object för att öka prestandan Görs ofta automatiskt av git vid en pull eller merge men kan vara bra att göra manuellt om repositoryt har blivit onödigt stort.
git fsck Kontrollerar alla objekt och referenser i ett repository, så att det inte finns några bristfälliga länkar eller ogiltiga objekt. Används om man vill kontrollera så att det inte finns några problem med integriteten eller konsistensen i ditt repository. Görs automatiskt av Git under vanliga operationer.
git fsck Kontrollerar alla objekt och referenser i ett repository, så att det inte finns några bristfälliga länkar eller ogiltiga objekt. Används om man vill kontrollera så att det inte finns några problem med integriteten eller konsistensen i ditt repository. Görs automatiskt av Git under vanliga operationer.
git commit --amend Öppnar ett textredigeringsprogram för att ändra det senaste commit-meddelandet och innehållet i commiten. Används om man glömt att inkludera någon fil eller ändring i den senaste commiten. Använd bara om du vet att ingen har clonat din commit. Annars kan det bli problem.
git help [command] Öppnar Git-manualen för att läsa dokumentationen om den önskade commanden Används om man vill läsa på om en specifik command
git cherry-pick [commit] Tar en specifik commit från en annan gren och applicerar den på den aktuella grenen Används för att välja en enskild commit från en annan gren och applicera den på den aktuella grenen. Det är användbart när du vill inkludera en specifik ändring från en annan gren utan att behöva sammanfoga hela grenen
git rebase [branch] Flyttar de lokala ändringarna till basen av en annan gren Används för att omorganisera commit-historiken genom att flytta dina lokala commits till basen av en annan gren. Det används ofta för att hålla commit-historiken ren och linjär särskilt vid arbete med grenar som har delats med andra.
git blame [fil] Visar vem som senast ändrade varje rad i en fil och vilken commit som gjorde ändringen Används för att spåra vem som gjorde varje ändring i en specifik fil genom att visa vilken commit som introducerade varje rad.
git grep [pattern] Söker efter mönster i filerna i arbetsträdet Används för att söka efter ett visst mönster i filerna i arbetsträdet. Detta är användbart för att snabbt hitta förekomster av specifikt innehåll i projektet.
git bisect [bad] [good] Används för binär sökning för att hitta en specifik commit som orsakade en förändring i beteendet mellan två tillstånd Används för att genomföra en binär sökning för att identifiera den specifika commit som orsakade en förändring i beteendet mellan två kända tillständ, en "dålig" och en "bra" commit.
git describe Ger en beskrivning av den närmaste taggen till en specifik commit. Ger en beskrivning av den närmaste taggen till en specifik commit, vilket kan vara användbart för att förstå var en viss commit befinner sig i förhållande till taggar i projektet.
git archive --format [zip|tar] [name] Skapar en arkivfil av en specifik gren med angivet format. Används för att skapa en arkivfil av en specifik gren med det angivna formatet (zip eller tar), vilket kan vara användbart för att skapa distributionspaket av en specifik gren.
git checkout -b [branchname] Skapar och växlar till en ny gren. Skapar en ny gren med det angivna namnet och växlar sedan till den nya grenen. Detta är en förkortad version av två separata kommandon: git branch [branchname] och git checkout [branchname].
git mergetool --tool [your_merge_tool] Öppnar ett verktyg för sammanfogning för att lösa konflikter i en sammanfogning. Öppnar ett verktyg för sammanfogning som är konfigurerat i din Git-konfiguration. Detta används vanligtvis för att hjälpa till att lösa konflikter som uppstår under sammanfogning av grenar.
git fetch origin Hämtar alla ändringar från fjärr-repot (origin) utan att integrera dem med den lokala grenen. Hämtar alla ändringar från fjärr-repot (origin) utan att integrera dem med den lokala grenen. Detta är användbart för att hålla den lokala grenen uppdaterad med förändringar från det fjärr-repot den är kopplad till.
git show --patch [commit] Visar detaljer om en specifik commit, inklusive skillnader i filinnehåll. Visar detaljer om en specifik commit, inklusive skillnader i filinnehåll. Alternativet --patch visar också de faktiska ändringarna som gjordes i varje fil.
git tag -a [tagname] [commit] Skapar en annoterad tagg för en specifik commit. Skapar en annoterad tagg för en specifik commit. Detta används för att markera specifika punkter i projektets historia, till exempel versionssläpp eller milstolpar.
git stash push -m "[message] Döljer lokala ändringar och sparar dem i en tillfällig "stash" med en meddelandebeskrivning. Döljer lokala ändringar och sparar dem i en tillfällig "stash" med en meddelandebeskrivning. Detta kan vara användbart när du behöver växla grenar eller utföra andra operationer utan att inkludera eller förstöra dina lokala ändringar.
git stash list Visar en lista över alla tillfälliga sparade ändringar (stash) Visar en lista över alla tillfälliga sparade ändringar (stash), vilket ger dig överblick över vilka ändringar som har sparats undan temporärt.
git rebase --interactive [branchname] Startar en interaktiv omgruppering där du kan redigera, kombinera eller ta bort commits när du omgrupperar mot en specifik gren. Startar en interaktiv omgruppering där du kan redigera, kombinera eller ta bort commits när du omgrupperar mot en specifik gren. Detta är användbart för att rensa upp commit-historiken och förbereda den för delning med andra.
git init Initierar ett nytt Git-repositorium i den aktuella mappen Används för att skapa ett nytt Git-repositorium från grunden. Använd detta kommando när du vill börja spåra ändringar i ett nytt projekt
git clone Klonar ett befintligt Git-repositorium från en angiven URL. Används för att skapa en kopia av ett befintligt Git-repositorium från en fjärrplats. Detta är användbart när du vill arbeta med en existerande kodbas.
git add Lägger till en specificerad fil i stegområdet Används för att inkludera ändringar i en specifik fil till det område som ska ingå i nästa commit. Detta kommando förbereder ändringarna för att registreras.
git commit -m Registrerar ändringar i repositoriet Används för att spara ändringar i Git-repositoriet som en ny commit. Med ett meddelande kan du beskriva syftet med ändringarna i commiten.
git status Visar den aktuella statusen för arbetsmappen och stegområdet Används för att visa den aktuella statusen för ditt arbetsområde och det område som är redo för att inkluderas i nästa commit. Detta ger dig översikt över ändringar som har gjorts.
git diff Visar skillnaderna mellan arbetsmappen och stegområdet Används för att visa skillnaderna mellan ändringar som finns i arbetsmappen och de som finns i stegområdet. Detta hjälper dig att se vilka ändringar som är redo att inkluderas i nästa commit.
git diff --staged Visar skillnaderna mellan stegområdet och den senaste commiten Används för att visa skillnaderna mellan ändringar som är redo att inkluderas i nästa commit (stegområdet) och den senaste commiten. Detta ger dig översikt över vad som kommer att ändras.
git branch Listar alla befintliga grenar i repositoriet Används för att visa en lista över alla git-grenar som finns i det aktuella Git-repositoriet. Det ger dig översikt över olika utvecklingslinjer.
git checkout [gren] Byter till den angivna grenen Används för att byta till en annan git-gren i ditt lokala repository. Detta används vanligtvis för att börja arbeta på en annan funktionsgren.
git merge [gren] Sammanfogar den angivna grenen med den aktuella grenen Används för att sammanfoga ändringar från en annan git-gren till den aktuella grenen. Detta används ofta för att integrera nya funktioner från en utvecklingsgren till huvudgrenen.
git pull Hämtar ändringar från det externa repositoriet och sammanfogar dem med den aktuella grenen Används för att hämta och integrera ändringar från ett externt Git-repositorium till den aktuella grenen i ditt lokala repository. Detta används vanligtvis när du behöver uppdatera din lokala kopia med de senaste ändringarna från ett fjärrplats.
git push Laddar upp lokala ändringar till det externa repositoriet Används för att ladda upp dina lokala ändringar till ett externt Git-repositorium. Detta gör att andra kan få tillgång till dina ändringar och bidra till projektet.
git remote Listar alla externa repositorier som är associerade med det lokala repositoriet. Används för att visa en lista över alla fjärrplatsreferenser som är associerade med det lokala Git-repositoriet. Detta ger dig översikt över fjärrplatsinställningar och URL:er.
git log Visar en kronologisk lista över commitar i den aktuella grenen. Visa en detaljerad logg över commitar i den aktuella Git-grenen.
git reset [fill] Avstår en specificerad fil från stegområdet. Återställ en specifik fil från stegområdet utan att påverka arbetsmappen.
